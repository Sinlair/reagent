  - Browser verification was not rerun here because no browser automation tool is available in this environment.
- US-003: Create workspace backup snapshots
  - Added a new top-level `reagent workspace snapshot` command.
  - Workspace snapshots now copy the full workspace directory plus the SQLite database file when `DATABASE_URL` uses a file-backed datasource.
  - Snapshot output writes a `manifest.json` with creation time, included content, workspace path, and database metadata.
  - Snapshot creation rejects overwrite attempts by default and refuses to create snapshots inside the active workspace directory.
- US-004: Preview workspace restore before applying it
  - Added `reagent workspace restore preview <snapshotPath>` to inspect a snapshot directory or its `manifest.json` before any write occurs.
  - Preview output shows snapshot creation time, target workspace path, database path or URL, and all included workspace/database items.
  - Restore preview is read-only and reuses the stored snapshot manifest instead of inferring state from the live workspace.
- US-005: Apply workspace restore safely
  - Added `reagent workspace restore apply <snapshotPath> --yes` so restore remains explicit and non-interactive.
  - Restore apply now refuses to run without `--yes` and creates a protection backup of the current workspace and current SQLite database before overwriting anything.
  - Restored workspace contents are copied back from the snapshot payload, and file-backed SQLite databases are restored from the snapshot when present.
  - Restore apply refuses to target the current working directory directly and keeps protection data outside the active workspace path.
- US-006: Export a support bundle without leaking secrets
  - Added `reagent workspace support-bundle` to export a local diagnostic bundle.
  - The bundle includes runtime status, a doctor-style summary, recent stdout and stderr tails, channel lifecycle audit, and agent runtime audit.
  - Bundle output writes a concrete local directory with `bundle.json` and excludes plaintext secrets by default because it does not embed raw `.env` or secret config values.
  - Support bundle creation rejects overwrite attempts by default through a unique bundle directory target.
- US-007: Add product-facing recovery hints to key surfaces
  - Expanded the existing product alert strip so it now appears on Landing, Overview, Channels, Agents, and Settings instead of only a subset of operational tabs.
  - Added explicit next-step recovery copy for fallback-only research mode, runtime service issues, channel setup blocking states, and unloaded agent runtime sessions.
  - Recovery hint copy now points users toward concrete commands such as `reagent onboard --apply`, `reagent runtime status`, `reagent doctor`, `reagent channels login`, and `reagent agent sessions`.
