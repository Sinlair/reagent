# Ralph Progress

Project: ReAgent
Branch: ralph/reagent-cognitive-kernel-v1
Started: 2026-04-18

Completed
- US-001: Define a structured neuron node model
  - Upgraded the internal neuron state from flat string arrays into structured nodes with `id`, `kind`, `content`, `salience`, `confidence`, `source`, and `updatedAt`.
  - Added support for `perception`, `memory`, `hypothesis`, `reasoning`, `action`, and `reflection` node kinds inside the persisted session digest.
  - Reworked neuron generation so new turns create structured cognition nodes rather than only string summaries.
- US-002: Persist cognition state alongside the legacy session digest
  - Kept `recentUserIntents`, `recentToolOutcomes`, and `pendingActions` as compatibility summaries while persisting the richer cognition state under `digest.neurons`.
  - Added backward-compatible session-store parsing so old string-array neuron data can still be read and upgraded into structured nodes.
  - Updated prompt injection to read the structured neuron model and render a richer `Neuron state` block.
- US-003: Align cognition memory with workspace memory and artifacts
  - Wired memory recall hits into digest generation so `memory` neurons can persist explicit `workspace-memory` and `artifact-memory` provenance alongside session and tool-derived context.
  - Kept long-term memory in the `memory` layer instead of leaking it into `hypothesis`, while preserving source metadata for each neuron node.
  - Expanded runtime digest coverage so tests now verify both workspace-backed and artifact-backed cognition memory sources.
- US-004: Support multiple hypotheses instead of a single implicit conclusion
  - Extended `hypothesis` neurons with optional `status`, `supportingEvidence`, and `conflictingEvidence` fields while keeping the canonical neuron shape backward-compatible.
  - Reworked hypothesis generation so the runtime can keep multiple simultaneous hypotheses alive, including source-backed workspace and artifact alternatives plus provisional follow-up hypotheses.
  - Updated prompt injection and session persistence so operators and the model see structured multi-hypothesis summaries without exposing private chain-of-thought.
- US-005: Add neuron activation, decay, and conflict updates
  - Added neuron-layer reconciliation so repeated judgments can be reactivated instead of being rebuilt from zero on every turn.
  - Introduced stale-node decay for carried-over `memory`, `hypothesis`, and `reasoning` layers so older cognition can fade over turns instead of remaining equally strong forever.
  - Tightened conflict handling so hypothesis confidence can drop or remain provisional when competing evidence stays active.
- US-006: Inject structured neuron summaries into prompts
  - The runtime now injects structured `Neuron state` blocks with salience, confidence, source, status, and evidence summaries into the model payload.
  - Prompt injection continues to coexist with memory primer and recent session history instead of flattening everything into one raw transcript.
  - The injected cognition view remains an operator-safe summary rather than exposing private chain-of-thought.
- US-010: Expose cognition inspection in the root CLI
  - Added `reagent agent cognition <sessionId|senderId>` with both human-readable and `--json` output.
  - CLI cognition output now surfaces `perception`, `memory`, `hypothesis`, `reasoning`, `action`, and `reflection` layers, plus supporting and conflicting evidence summaries.
- US-011: Expose cognition inspection in the HTTP API
  - Added `/api/agent/sessions/:sessionId/cognition` on the canonical agent API surface.
  - The API returns the same structured cognition model used by the CLI, without exposing raw private chain-of-thought.
- US-012: Add a cognition panel to the Web Agents surface
  - Added a `Cognition` tab to the Web `Agents` panel and wired it to the new cognition API.
  - The panel now renders empty, provisional, and conflict-heavy cognition layers in one operator-facing view.
- US-007: Generate a reflection layer after each completed turn
  - Reworked `reflection` neurons so each turn can now record what was confirmed, what remains uncertain, and the recommended next action.
  - Reflection summaries now inherit supporting and conflicting evidence from the active hypothesis state instead of only emitting one generic sentence.
- US-008: Drive tool choice from cognition state
  - Added cognition-driven tool guidance to tool-turn instructions so the model reads `hypothesis`, `reasoning`, and `action` layers before selecting a tool.
  - Introduced a runtime posture model that biases the agent toward evidence-gathering or delivery-ready tools based on current cognition state.
  - Added policy enforcement that blocks premature synthesis or delivery tools when the cognition layer still indicates unresolved uncertainty.
  - Extended runtime policy to respect entry source and role, so constrained channel entries and lightweight assistant sessions do not jump into long-form delivery tools too early.
- US-009: Drive bounded delegations from cognition state
  - Delegation creation now derives a cognition-backed rationale from the current `action` and `hypothesis` layers, including posture, matched signals, and recommended delegation kinds.
  - The agent API blocks misaligned `synthesis` delegations when cognition still prefers evidence gathering, unless the prompt explicitly asks for that deliverable.
  - Delegation lifecycle reads now sync delegation status back into session cognition so delegation progress can become part of the agent's durable memory and next-action state.
  - CLI and Web delegation views now surface the cognition-backed rationale, posture, and matched action/hypothesis signals for each delegation.
  - Delegation lifecycle policy now blocks duplicate active delegations for the same task/kind and prevents synthesis while evidence delegations for that task are still running.
  - Recent failed delegations now enter a retry cooldown window; retrying them requires an explicit retry-style prompt and the cognition layer records the follow-up recommendation.
  - Delegation records now expose explicit retry metadata (`retryState`, `retryAfter`, `retryHint`) so operators can see retry timing directly in CLI and Web surfaces.
