# ReAgent v0.1.10

Command-first chat routing release.

## Highlights

- OpenClaw-inspired command-first entry flow
  - slash commands are intercepted first
  - plain chat and workspace-heavy requests now split before runtime dispatch
- plain chat path
  - lightweight greetings and normal chat now use direct text generation instead of the tool loop
- better runtime diagnostics
  - tool-runtime failures are logged before fallback so upstream gateway issues are visible

## Package

- `@sinlair/reagent`
  - standalone always-on runtime and CLI

## Validation

Validated for this release with:

- `npm.cmd run release:verify`
- `node tests/platform.test.mjs`
- `node tests/provider.test.mjs`
- manual runtime check via `reagent.cmd channels chat wx-debug "你好" --json`

## Recommended Release Notes Summary

ReAgent v0.1.10 improves channel chat behavior by adopting a more OpenClaw-like command-first routing model. Normal greetings and lightweight chat now behave like regular conversation, while research and workspace-heavy messages still route through the agent runtime.
