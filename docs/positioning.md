# Positioning: ReAgent vs Hermes-Agent

ReAgent is not a direct replacement for Hermes-Agent.

## What ReAgent Is

ReAgent is a:

- research-first workspace
- local-first runtime
- durable artifact and memory system

It prioritizes:

- structured briefs
- discovery quality
- evidence-backed reports
- durable workstreams and delivery artifacts

## What ReAgent Borrows From Hermes-Agent

ReAgent intentionally borrows Hermes-style runtime seams such as:

- tool registry
- runtime hooks
- audit trails
- profile and session inspection
- progressive skill disclosure
- bounded delegation

## What ReAgent Does Not Claim

ReAgent does not currently claim to be:

- a completed general-purpose agent runtime
- a fully generic multi-channel host platform
- a drop-in Hermes-Agent clone

## Public Positioning Rule

When describing ReAgent publicly:

- describe it as a research workspace with Hermes-style runtime seams
- do not market it as “Hermes-Agent compatible in every layer”
- do not market it as a finished general-purpose agent platform
